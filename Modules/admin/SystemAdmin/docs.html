Admax User Interface - Documentation

Thank you for choosing Admax!

Feedback and suggestions are welcome and will be considered for future updates. You can leave them as a comment on the item's page, or send me an email through my profile page.

I recommend inspecting Admax with your browser's Dev Tools as you're reading the documentation, to better understand its structure and code, and get a visual representation.

File structure

  • css/ - stylesheet files
    • anims.css - CSS3 animations & transitions
    • core.css - reset / normalize, grid, font-faces
    • global.css - helper classes, styles and colors
    • icons.css - WebHostingHub Glyphs font icons
    • inputs.css - input styling
    • login.css - login stylesheet
    • style.css - main styling and layout
    • ui.css - UI elements styling
  • img/ - the image directory, of course
  • js/ - jQuery files and plugins
    • flot.js - Flot chart plotting plugin
    • functions.js - global functions and lightweight plugins
    • inputs.js - input functions like checkbox, form validation, tooltips, notifications and more
    • jquery.js, jquery-ui.js - jQuery and jQuery UI files
    • justgage.js - gage plugin
    • sparkline.js - sparkline chart plugin
    • tablesorter.js - table filtering plugin
  • res/ - resources like fonts, the PHP files and plugins like elFinder and tinyMce
  • 403, 404, 503.html - error pages, the bad news givers of the internet
  • analytics.html - lines, bars, pies, donuts (not the delicious kind, though)
  • dash.html - the homepage, the first page you see after logging in or hacking an account
  • extras.html - extras page, growl alerts, typography, file manager
  • grid.html - grid demo - sizes, combinations, nesting
  • inputs.html - the inputs page - buttons, selects, tooltips, forms and various fields
  • ui.html - UI elements page - accordion, sortable boxes, notifications and more

The smaller plugin stylesheets have been included in ui.css, minified plugins have been included in functions.js and inputs.js. Larger plugins and stylesheets are standalone.

HTML

Basic page structure

<div id="wrapper"> - wraps all the page's content

<div id="sidebar"> - the sidebar, which includes the navigation, user avatar and notifications
Can set position by adding class sidebar-hz sidebar-top, sidebar-hz sidebar-bottom or sidebar-right to <div id="wrapper">.

<div id="nav"> - navigation menu. 

	        <ul id="nav">
	                <li class="active"><span class="nav-icon icon-home"></span>Active menu item</li>
	                <li>
			                <a href="link to page.html"></a><span class="nav-icon icon-chart"></span>Basic menu item
			                <ul class="sub-nav">
					                <li><a href="link to page.html"></a><span class="nav-icon icon-name"></span>Sub-menu item</li>
			                </ul>
				    </li>
	        </ul>

<div id="content"> - the page's content goes here. Sometimes has a page class, so content on that page can be easily arranged or styled.

<div class="modal-ov"> - the modal window container and overlay. Visible with class show.

</div> - end wrapper.

Boxes

Also known as content holders, panels, or "innard restrictors", boxes are containers for elements, text or what have you. There are 2 main types: simple, like this one and with header.

		<div class="box">I'm a simple box</div>
Box with header
			    <div class="box">
			        <div class="header">I'm a header</div>
			        <div class="content">
			            And I'm the body!
			        </div>
			    </div>

Clicking on a box's header will collapse it. Adding class coll will make it collapsed by default.

Box collapsed by default
And now it's not collapsed anymore.

To add an icon to a box header, just append <span class="icon icon-name"></span> to it. The same for buttons and button groups (code in the "Inputs" section).

If you want it on the right, add class right to it.

Content navigation

The tab-like content navigation is useful when you need to display a lot of information in a restricted space, or you just want to have tabs because you are an adult and can do whatever you want.

The selected tab gets class sel and its corresponding content gets class show. 

			<div class="box nav-box">
			    <ul class="nav">
			        <li data-nav="#tab1" class="sel">Tab 1 (selected)</li>
			        <li data-nav="#tab2">Tab 2</li>
			    </ul>
			    <div class="nav-cont">
			        <div id="tab1" class="nav-item show">Tab 1 content (visible).</div>
			        <div id="tab2" class="nav-item">Tab 2 content.</div>
			    </div>
			</div>

Adding class nav-big to the nav box will enlarge the nav.

Classes left, right and btm set the nav's position.

To make the nav dark, add class dark to .nav.

Content is transitioned vertically or horizontally. The default transition direction is horizontal, to make it vertical add class vt to the nav box.

Modal dialogs

A modal dialog's structure is similar to the box. The code below is for a modal with header, body and navigation menu, like the one you can see on every page if you click Settings or User CP.

			<div class="modal-ov">
			    <div class="box modal" id="mod-id">
			        <div class="header">Modal Title</div>
			        <div class="mod-body">
			            Modal content
			        </div>
			        <div class="mod-act">
			            <button class="close">Close</button>
			       </div>
			     </div>
			</div>

To show a modal on click, assign an id to the element you want to trigger it and add a data-modal attribute with the id of the target modal. Then, add class mod-tg to the trigger or call the modal function with $('#trigger-id').modal();.

If the modal's content is tabbed and you want the trigger to show a specific tab, also add the tab id in the data-modal attribute. 

<button id="trigger-id" data-modal="#mod-id #tab-id">Show modal</button>

When a modal is triggered, both the modal container (.modal-ov) and the triggered modal get class show. Clicking on the modal overlay or a button with class close will hide the modal.

Icons

To display an icon, add class icon icon-name to an element.

For arrows use class arrow instead.

You can see the arrow character map in "Arrows.jpg".

To browse the icons, go to WebHostingHub's website. 

			<span class="icon icon-home"></span>

Other elements

Tooltips

			<button class="ttip" title="The tooltip will display this">Show tooltip</button>

Tooltips can be triggered from any element, it just needs class ttip and a title or data-ttip="What you want the tooltip to say" attribute.

Options can be set with the data-ttip-opt attribute, get added to the tooltip element as classes, and can be the following:

  • top - positions tooltip to the top of the element (default).
  • left - positions tooltip to the left of the element.
  • right - positions tooltip to the right of the element.
  • bottom - positions tooltip to the bottom of the element.
  • dark - dark tooltip instead of light.

jQuery UI's "position" function is used to perfectly align the tooltip to the element, so jquery-ui.js must be included on pages with tooltips.

Notifications

			<div class="notif">
			    Notification text <span class="icon">Icon</span>
			    <p class="nt-det">Notification details</p>
			</div>

If a notification contains details (.nt-det) it will expand on first click and close on second. If it doesn't contain details, it will close on first click.

  • hide - hides notification.
  • auto - sets height to auto.
  • expanded - expands notification, displaying the details.
  • no-icon - removes extra padding on the right.
  • $.fn.notif(); - included in $.fn.inputs();

Growl notifications

The growl notifications use toastr. Simple and easy to use, you can create a growl style notification with: 

			toastr.success('Success growl');

There are 4 types of growls: success, error, warning and info. You can also give it a title, and set options like position and timeout. The code below demonstrates how to show a growl when a button is clicked: 

			$('#btn').click( function() {
			    toastr.options.positionClass = 'toast-top-left';
			    toastr.error('Shutting down in... now' , 'Error');
			});

The position options are:

toast-top-right (default), toast-top-left, toast-bottom-right, toast-bottom-left, toast-top-full and toast-bottom-full.

Sliders

			<div id="slider" class="slider"></div>

			$('#slider').slider();

To enable the value/grade display, call the $('#slider').gradeslider(); function.

The grid

Admax uses a simple & straight-forward grid system. There are 16 sizes (g1 to g16), so two boxes with class g8 would each be half the content size and add up to 100% width, forming a row.

If you don't want the element to have margins, use class ig1 to ig16 instead. Both .ig16 and .full wil set the width to 100%.

See the grid page to see each grid size and how wide it would be on your screen.

Buttons

There are 4 button sizes: btn-s small (24px), normal (28px), btn-m medium (30px) and btn-l large (34px).

Adding an icon to a button is the same as with a box header. Append an icon element to it, and it's done.

For buttons that only contain the icon and no text, add class no-mg to the icon element.

You can adjust the icon using the font-size and line-height helper classes (detailed in the CSS section). Hover over the blue button to see the code.

Adding class toggle will make the button toggleable. Adding class pressed will activate the pressed state.

Button groups

Button groups are very easy to create, just make a div with class buttons and append the buttons you want grouped to it.

You can set each button's width using the 'igX' grid classes.

You can add button size classes directly to the .buttons div instead of on each button separately. Adding class wide will make the buttons wider.

Adding class btn-radio to a button group will "radio" the buttons. 

	        <div class="buttons btn-m">
	            <button>Previous</button>
	            <button>Next</button>
	        </div>
	        <div class="buttons btn-radio btn-l">
	            <button>This</button>
	            <button>No, this</button>
	        </div>

Input groups

You can group multiple inputs using an input group. As with the button groups, you can set the width on each element using 'igX' classes. 

	        <div class="input-group">
	            <input type="text" class="ig5" value="Input 1">
	            <input type="password" class="ig4" value="Input 2">
	            <input type="text" class="ig3" value="Input 3">
	            <input type="text" class="ig4" value="Input 4">
	        </div>

All inputs are shown on the inputs page, so check it out if you haven't yet.

  • inset - inset appearance
  • valid - valid input
  • error - invalid input, red border is added

Spinner

  • big - big buttons

Drops

There are 2 types of drops: single button or split button.

				<div class="drop single">
					<button class="btn-m black">Dropdown <span class="arrow">&</span></button>
					<ul>
						<li>Option 1</li>
						<li>Option 2</li>
					</ul>
				</div>
  • Option 1
  • Option 2

				<div class="drop">
					<button class="btn-m black">Dropdown</button>
					<button class="btn-m black arrow">&</button>
					<ul>...</ul>
				</div>
  • Option 1
  • Option 2
  • al-r - aligns the drop menu to the right of the button
  • arrow-l - moves arrow to the left
  • drop-l - sets drop direction to the left
  • drop-r - sets drop direction to the right
  • drop-u - sets drop direction upward
  • $.fn.btnDrop(); - included in $.fn.inputs();

Direction classes can be combined for more drop directions. For instance, drop-l drop-u will set the drop direction up and to the left.

Select

				<select>
					<option>Select</option>
					<option>An</option>
					<option>Option</option>
				</select>

Selects are transformed into the following HTML structure, to allow full customization:

				<div class="drop select">
					<select style="display: none;">...</select>
					<ul>
						<li class="sel">Select</li>
						<li>An</li>
						<li>Option</li>
					</ul>
					<span class="opt-sel">Select</span>
					<span class="arrow">&</span>
				</div>

<span class="opt-sel"> displays the selected option's value. If no option is selected, it gets the value of the first option.

When an option is selected, it gets the selected attribute added and its corresponding <li> gets class sel.

  • multiple - allows for multiple options to be selected, listed as tags
  • no-arrow - the arrow will not be added to the HTML
  • has-delete - adds clear button for resetting the select
  • has-search - adds search input
  • disabled - disables transformed select, gets added if the select has attribute 'disabled'.
  • $.fn.makeSelect(); - transforms selects into the custom HTML structure, included in $.fn.inputs();
  • $.fn.selectfn(); - provides functionality for transformed selects, called by $.fn.makeSelect(); , included in $.fn.inputs();

Checkboxes

				<input type="checkbox" class="checkbox">
				<input type="checkbox" class="checkbox big">
Like selects, checkboxes get transformed into the following HTML structure: 
				<div class="checkbox-cont checkbox unchecked">
					<input type="checkbox" class="chbox" style="display: none;">
					<div class="on"><span class="icon-ok"></span></div>
				</div>
  • disabled - disables transformed checkbox, gets added if the checkbox has attribute 'disabled'.
  • disabled - disables transformed checkbox, gets added if the checkbox has attribute 'disabled'.
  • $('.checkbox, .radio').checkfn(); - included in $.fn.inputs();

Form validation

Admax's form validation is based on jQuery Tiny Validation, but most of the code has been written from scratch.

The inputs that can be validated as required are text inputs, selects, checkboxes and spinners. Text input validation options include e-mail or matching fields (for passwords, etc), and can be set a minimum number of characters with the data-min="x" attribute.

Each input type has a custom error message, displayed in a neat tooltip.

To validate a form, assign an id to it and add one of the classes below to the inputs in the form you want validated.
Then, call the validate function $('#form-id').validate();.

  • required - makes input required, form will not submit until it is valid.
  • email - requires a valid e-mail address format (xyz@xyz.xyz) to be entered.
  • match - requires two inputs to match (each must have this class, and a data-match="same-for-every-input-in-set" attribute.

CSS

This section will explain the CSS files and the properties they contain.

The CSS code is organized by elements and each part is commented, so finding and customizing what you need should be a piece of cake.

Styles for the dark color scheme begin with '.dark ', and are usually located at the end of the element's styling for accessibility.

anims.css

Contains the CSS3 animations and transitions. Each one is commented, and the names are self-explanatory.

View a demo on the author's site: CSS3 animations.

core.css

Contains reset/normalize code, page load spinner, font-faces and the grid

global.css

Contains helper classes like floats, margins, padding, border-radius, gradients and colors. The code is pretty much self explanatory, reading through it should give you an idea of what each class does.

icons.css

Stylesheet for the WebHostingHub Glyphs icon font.

inputs.css

Contains styles for buttons, checkboxes, inputs, textareas, dropdowns, selects, spinner inputs, input and button groups, and input error tips.

login.css

Styling for the login page, user select/switch, username not found prompt and register form.

style.css

This is the main styling and layout file.

It contains styling for each page's underlying structure, the sidebar, boxes, all page specific properties and responsive code.

ui.css

Contains the styling for the top bar, navigation menus and wizard, accordion, notifications, tooltips, modals, sliders and other UI elements and widgets.

jQuery functions

functions.js

  • $.fn.tgclass() - toggles a class on the target on click. Trigger needs a data-tgcls="#target class" attribute. Automatically called on elements with class tgcls in $.fn.loadfns();
  • $.fn.loadfns() - the page load and global calls function. If a page has functions that must be called on load, include them in the parameter as a function.
			$.fn.loadfns( function() {
			  $.fn.callOnLoad(); // This function will be called on document load
			};

inputs.js

  • $.fn.checkfn() - the checkbox function, included in $.fn.inputs();
  • $.fn.notif() - notification function, included in $.fn.inputs();
  • $.fn.scrollspy() - scrollspy function, as seen on the inputs and documentation pages.
  • $.fn.tooltip() - tooltip function, automatically called on elements with class ttip in $.fn.loadfns();
  • $.fn.btnDrop() - drop function for selects and drops, included in $.fn.inputs();
  • $.fn.selectfn() - function for selects and multiple selects, included in $.fn.inputs();
  • $.fn.makeSelect() - function for transforming selects into the custom HTML structure, included in $.fn.inputs();
  • $.fn.closeOnBlur() - function for closing selects and drops when clicking outside or on another select/drop
  • $.fn.validate() - function for validating forms. Has a function for each input type
  • $.fn.numberInp() - function for number only inputs (class number), included in $.fn.inputs();
  • $.fn.fileSel() - function for file upload inputs, included in $.fn.inputs();
  • $.fn.gradeslider() - function for sliders with value/grade display
  • $.fn.inputs() - global function containing frequently used input function calls, and the input delete (clear) function
